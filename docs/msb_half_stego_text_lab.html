Stego Text Lab — LSB/MSB, payload e secondo livello

Laboratorio locale per cercare testo in immagini steganografiche. Priorità ai bit 0-3 RGB quando i 4 LSB contengono già una seconda immagine; MSB half mantenuto come controllo secondario.

Usa il file originale, non screenshot o esportazioni intermedie. L'elaborazione resta nel browser.

Nel tuo caso i bit 4-7 sembrano cover. Il preset consigliato è “Priorità payload”.

Trim trailing bits

Usa “Italiano compatto” se il messaggio atteso è una stringa italiana continua, ad esempio questoeunmessaggio..., senza spazi.

Per JPG/PNG/ZIP il lab prova a individuare anche il fine file. Per GZIP/ZLIB non esiste un marker di fine semplice e affidabile: viene proposto il blocco fino al prossimo magic byte o a fine stream.

Usato nello scan automatico insieme a magic bytes. In modalità “Italiano compatto” puoi inserire parole italiane senza spazi: verranno cercate anche dentro stringhe continue.

Strategia consigliata: 1) file originale; 2) bit 0-3 RGB; 3) singoli bitplane 0/1/2/3 per R,G,B; 4) magic bytes; 5) solo dopo controllo MSB 4-7.

Usa il pulsante “Anteprima” nei risultati dello scan oppure “Visualizza blocco come immagine/PDF”.

Se dai 4 bit meno significativi hai già ricavato una seconda immagine, allora è molto probabile che i bit 0-3 contengano il payload principale. I bit 4-7 sono invece la parte più significativa del pixel della cover e quindi mostrano ancora strutture visive naturali.

La corrispondenza tipica è:

stego bit 3 → hidden bit 7stego bit 2 → hidden bit 6stego bit 1 → hidden bit 5stego bit 0 → hidden bit 4

Per questo il lab ora dà priorità a: singoli LSB plane, combinazione dei 4 LSB, ordini canale alternativi, offset bit e ricerca di magic bytes.
